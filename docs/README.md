# 📚 The docs — what to read, and in what order

Two layers live in this folder. **Beginner docs** guide you while you learn;
**reference docs** are for depth and interview prep later. You never need to read
everything — use the table below to grab the right doc at the right moment.

## Layer 1 — Beginner guides (easy language, read in this order)

| # | Doc | Read it when… |
|---|---|---|
| 1 | [START_HERE.md](START_HERE.md) | you've got the app running and wonder "now what?" — the study plan |
| 2 | [HOW_THE_APP_WORKS.md](HOW_THE_APP_WORKS.md) | you want to understand how the project's files connect (week 3-ish, or when curious) |
| 3 | [GLOSSARY.md](GLOSSARY.md) | any word confuses you — keep it open in a tab, always |
| 4 | [EXERCISES.md](EXERCISES.md) | you finish studying a concept — 2–3 small tasks per concept |
| 5 | [DEBUGGING.md](DEBUGGING.md) | something broke, or you want to install React DevTools (do it week 1) |
| 6 | [BUILD_A_FEATURE.md](BUILD_A_FEATURE.md) | you finish the concepts — build a full page yourself, like a real work ticket |

## Layer 2 — Reference & interview prep (denser, for later)

| Doc | What it's for |
|---|---|
| [ANGULAR_TO_REACT.md](ANGULAR_TO_REACT.md) | coming from Angular? the translation dictionary — useful from day one |
| [HOOKS_CHEATSHEET.md](HOOKS_CHEATSHEET.md) | every hook on one page — quick syntax reminders |
| [TESTING.md](TESTING.md) | how React testing works (pairs with `src/__tests__/`) |
| [INTERVIEW_QUESTIONS.md](INTERVIEW_QUESTIONS.md) | 52 Q&A + 10 coding challenges — when an interview approaches |
| [RENDERING_AND_RECONCILIATION.md](RENDERING_AND_RECONCILIATION.md) | virtual DOM, Fiber, keys — the senior-round deep dive |
| [TYPESCRIPT_WITH_REACT.md](TYPESCRIPT_WITH_REACT.md) | when you're ready to add types (a later exercise) |
| [ECOSYSTEM_AND_BEYOND.md](ECOSYSTEM_AND_BEYOND.md) | Next.js, TanStack Query, the wider landscape + the planned Next.js exercise |

## Where the rest of the learning material lives

- **The app itself** — `npm run dev`, it opens on an in-app "Start Here" page.
- **`src/concepts/<nn-name>/`** — each concept's commented code (`.jsx` with a
  📖 Beginner's Map at the top) + its `NOTES.md` (theory & interview questions).
- **`README.md`** (repo root) — setup from zero, including installing Node.js.
